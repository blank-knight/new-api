# Vibe Coding 核心规则 — API 中转站

## 项目本质
这不是从零写代码的项目，而是**配置+部署+运营**的项目。
New API 已经是成熟的开源产品，我们的工作是：
1. 正确部署和配置
2. 对接上游供应商
3. 定制定价和运营策略
4. 监控和运维

## 修改原则
- **配置优于代码**：尽量通过环境变量和后台管理修改，不改源码
- **最小改动**：只在必要时修改代码（如自定义支付、品牌定制）
- **记录配置**：所有配置变更记录在此项目中
- **升级兼容**：保留自定义配置的升级路径

## 不改源码的情况
- 添加渠道 → 后台管理界面操作
- 修改定价 → 后台管理界面操作
- 用户管理 → 后台管理界面操作
- 域名/SSL → Nginx 配置

## 需要改源码的情况
- 品牌定制（Logo、名称、域名）
- 自定义支付网关
- 内容审核增强
- 特殊限流逻辑

## 文件结构
```
new-api/
├── program-design-document.md   ← 项目设计文档
├── strategy-research.md         ← 策略研究
├── tech-stack.md                ← 技术栈
├── implementation-plan.md       ← 实施计划
├── vibe-coding-core.md          ← 本文件
├── docker-compose.yml           ← Docker 编排（修改密码等）
├── .env.example                 ← 环境变量参考
├── memory-bank/                 ← 运营记录
│   ├── progress.md              ← 进度追踪
│   └── operations-log.md        ← 运营日志
└── ...（New API 原有文件不动）
```

## 安全注意事项
- 所有密码使用强随机字符串
- 不在代码中硬编码 API Key
- 服务器只开放必要端口
- 定期备份数据库
- 监控异常访问
