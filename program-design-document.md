# API 中转站 — 项目设计文档

## 1. 项目概述

### 商业模式
**AI API 中转站**：批量采购各家 AI 厂商的 API 额度，通过 New API 网关统一对外提供接口，按 token/次收费，赚取差价。

```
上游供应商（批量采购，价格低）
    OpenAI / Claude / Gemini / DeepSeek / 通义千问 / ...
         │
         ▼
    New API 网关（本系统）
    - 统一 API 接口（OpenAI 兼容格式）
    - 用户管理 + API Key 分发
    - 计费 + 限流 + 监控
         │
         ▼
    下游用户（按量付费，价格高）
    - 开发者
    - 企业用户
    - AI 应用（ChatBot、Agent 等）
```

### 盈利模式
- **差价**：上游批量价 < 下游零售价
- **汇率差**：海外 API 用人民币结算（用户免外汇）
- **便利性溢价**：统一接口，用户不用分别对接各家
- **稳定性溢价**：自动 failover、负载均衡

### 目标用户
1. 国内个人开发者 — 需要海外 AI API 但没有海外支付方式
2. 小型 AI 应用团队 — 不想分别对接多家 API
3. 学生/研究者 — 需要低成本试用各种模型

## 4. 定价策略

### 原则
- 热门模型（GPT-4o、Claude）毛利低，走量
- 国产模型（DeepSeek、Qwen）毛利高，走性价比
- 新模型首发快速跟进，吸引开发者

### 阶梯定价（按用户等级）
| 等级 | 条件 | 折扣 | 说明 |
|---|---|---|---|
| 免费试用 | 注册即送 | 原价 | 送 $0.5 额度 |
| 普通用户 | 充值 < ¥500 | 原价 | 标准定价 |
| VIP | 充值 ≥ ¥500 | 9折 | 月付用户 |
| SVIP | 充值 ≥ ¥5000 | 8折 | 企业/大户 |

## 5. 部署架构

```
用户请求
   │
   ▼
[Nginx / Cloudflare] ← SSL + 域名 + CDN
   │
   ▼
[New API 服务] ← Docker 容器，端口 3000
   │
   ├──→ [PostgreSQL] ← 用户数据、计费、日志
   ├──→ [Redis] ← 缓存、限流、会话
   └──→ [上游 AI 供应商] ← 通过 HTTPS 调用
```

### 服务器要求
- **最低配置**：2C 4G（初期几十个用户）
- **推荐配置**：4C 8G（几百用户）
- **存储**：20G SSD 足够（数据库不存模型数据）
- **带宽**：API 中转主要是文本，1-5Mbps 起步
- **地域**：推荐香港/新加坡（同时覆盖国内外）

## 6. 域名与品牌
- 需要一个域名（面向用户的 API 地址）
- SSL 证书（Let's Encrypt 免费或 Cloudflare 托管）
- 管理后台可通过子域名访问（如 admin.xxx.com）

## 7. 风险与合规
- ⚠️ 国内提供 AI API 需要关注《生成式 AI 服务管理暂行办法》
- ⚠️ 需要内容审核/过滤机制
- ⚠️ 上游供应商 TOS 限制（部分禁止转售）
- ⚠️ 支付合规（需要企业支付宝/微信支付或 Stripe）
