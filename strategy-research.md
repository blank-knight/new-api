# 策略研究文档 — API 中转站

## 1. 市场分析

### 竞品调研

| 平台 | 特点 | 价格水平 | 用户量 |
|---|---|---|---|
| OpenRouter | 最知名的 API 聚合 | 接近官方价 | 大 |
| 国内各种中转站 | 便宜但不稳定 | 官方价 × 0.5-1.5 | 中小 |
| 各云厂商 API 市场 | 官方渠道 | 官方价 | 大 |
| 自建 New API 站点 | 开源方案，自己运营 | 自定 | 取决于运营 |

### 市场机会
1. **国内用户痛点**：没有海外信用卡 → 买不了 OpenAI/Claude API
2. **统一接口需求**：开发者不想对接 10 家 API，一个地址搞定
3. **稳定性需求**：官方 API 偶尔挂，中转站可以自动切换到备用渠道
4. **价格敏感**：中小用户只想花几块钱试用，官方最低充值太高

## 2. 采购策略

### 渠道选择优先级
1. **官方直采**（最优）— 价格透明，稳定性最好
   - OpenAI: 按量付费，预充值有折扣
   - Anthropic: 按 token 计费
   - Google: Vertex AI 有免费额度

2. **云厂商代理**（备选）
   - 阿里云百炼平台 — 通义千问 + 部分海外模型
   - 腾讯云 TI — 混元 + 部分海外模型
   - 硅基流动 (SiliconFlow) — 多模型聚合，价格低

3. **Azure/AWS**（企业级）
   - Azure OpenAI — 国内合规使用 GPT-4
   - AWS Bedrock — Claude/Gemini/Titan

### 成本优化
- **缓存命中**：相同 prompt 不重复调用上游（New API 内置支持）
- **模型路由**：简单任务用便宜模型，复杂任务用贵模型
- **批量采购**：DeepSeek 等国产模型可以包月/预购
- **多渠道竞价**：同一模型接入多个渠道，自动选最便宜的

## 3. 技术策略

### 高可用
```
用户请求 → New API
              ├── 渠道1 (OpenAI 官方)    ← 主渠道
              ├── 渠道2 (Azure OpenAI)   ← 备用
              └── 渠道3 (其他代理)       ← 兜底
```
- 同一模型至少 2 个渠道
- 自动健康检查 + failover
- 延迟监控，自动切换到最快渠道

### 限流策略
- 按用户限流（防止滥用）
- 按模型限流（热门模型排队）
- 按渠道限流（上游 RPM/TPM 限制）

### 监控指标
- 各渠道成功率/延迟
- 每日调用量/收入
- 用户留存率
- 成本/利润比

## 4. 运营策略

### 冷启动
1. **免费额度**：注册送 $0.5，降低试用门槛
2. **开发者社区**：在 V2EX、掘金、GitHub 推广
3. **文档优先**：写好 API 文档 + 示例代码
4. **微信群/Discord**：用户交流 + 实时支持

### 增长
1. **推荐返利**：老用户邀请新用户，双方各得 $1
2. **博客/教程**：写 AI 开发教程，引流到 API 站
3. **GitHub 项目集成**：开源项目默认推荐你的 API
4. **企业定制**：大客户可以谈私有部署/专属渠道

### 留存
1. **稳定性** — 这是最重要的，服务不能挂
2. **响应速度** — 延迟低，用户体验好
3. **价格竞争力** — 定期调研竞品价格
4. **客服响应** — 问题快速解决

## 5. 风险评估

| 风险 | 概率 | 影响 | 应对 |
|---|---|---|---|
| 上游封禁 | 中 | 高 | 多渠道冗余 + 低调运营 |
| 合规问题 | 中 | 高 | 内容过滤 + 用户协议 |
| 价格战 | 高 | 中 | 差异化服务 + 稳定性 |
| 技术故障 | 低 | 高 | 自动监控 + 快速切换 |
| 用户滥用 | 中 | 中 | 限流 + 审计 + 黑名单 |
